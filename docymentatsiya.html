📋 Обзор системы "СУГДОТЕЛ"

СУГДОТЕЛ - это современная система управления отелем с веб-интерфейсом, мобильным приложением и AI-ассистентом. Система предоставляет полный цикл услуг: от бронирования номеров до управления бронированиями и взаимодействия с гостями.

🎯 Ключевые преимущества

  • 🤖 Умный AI-ассистент на базе DeepSeek
  • 📱 Нативное мобильное приложение на Kotlin
  • ⚡ Быстрый веб-интерфейс на FastAPI
  • 📧 Автоматические email уведомления
  • 🔐 Безопасная аутентификация
  • 📊 Панель управления для администраторов

🚀 Установка и настройка

📋 Системные требования

Минимальные требования

  • ОС: Windows 10, macOS 10.14+, Ubuntu 18.04+
  • Python: 3.8+
  • Память: 2GB RAM
  • Место на диске: 500MB
  • Интернет: Для работы AI ассистента и email

⚙️ Настройка бэкенда (FastAPI)

1. Клонирование репозитория

# Клонируйте проект git clone https://github.com/your-username/sugdhotel.git cd sugdhotel # Создайте виртуальное окружение python -m venv venv source venv/bin/activate # Linux/Mac # venv\Scripts\activate # Windows

2. Установка зависимостей

# Установите необходимые пакеты pip install fastapi uvicorn python-multipart pip install sqlite3 requests jinja2 # Или используйте requirements.txt pip install -r requirements.txt

3. Настройка конфигурации

# В файле main.py укажите ваши настройки: # DeepSeek API ключ DEEPSEEK_API_KEY = "sk-your-api-key-here" # Email настройки EMAIL_ADDRESS = "your-email@gmail.com" EMAIL_PASSWORD = "your-app-password" # Запуск сервера uvicorn.run(app, host="0.0.0.0", port=8000)

4. Запуск сервера

# Запустите сервер python main.py # Или используйте uvicorn напрямую uvicorn main:app --host 0.0.0.0 --port 8000 --reload

Сервер будет доступен по адресу: http://localhost:8000

🎨 Настройка фронтенда

Фронтенд уже включен в бэкенд и обслуживается FastAPI. Дополнительная настройка не требуется.

⚠️ Важные замечания

  • Убедитесь, что статические файлы находятся в папке static/
  • Шаблоны должны быть в папке templates/
  • Для production используйте --reload только в development

📅 API Бронирований

POST /book

Описание: Создать новое бронирование

// Request Body (FormData) room_id: 1 guest_name: "Иван Иванов" guest_email: "ivan@example.com" guest_phone: "+992123456789" check_in: "2024-01-15" check_out: "2024-01-20" // Response Redirect to /booking-success/{booking_id}

POST /api/cancel-booking

Описание: Отменить бронирование

// Request Body (FormData) booking_id: 123 email: "ivan@example.com" // Response { "status": "success", "message": "Бронирование отменено" }

🤖 AI Ассистент API

WebSocket /ws/chat

// Подключение const ws = new WebSocket('ws://localhost:8000/ws/chat'); // Отправка сообщения ws.send(JSON.stringify({ message: "Привет", email: "user@example.com" })); // Получение ответа ws.onmessage = (event) => { const data = JSON.parse(event.data); console.log(data); };

📱 Мобильное приложение

🤖 Настройка Android приложения

1. Требования

  • Android Studio Arctic Fox или новее
  • Android SDK 21+ (Android 5.0)
  • Kotlin 1.5+
  • JDK 11+

2. Создание проекта

// app/build.gradle.kts plugins { id("com.android.application") id("org.jetbrains.kotlin.android") } android { compileSdk = 34 defaultConfig { applicationId = "com.sugddelux.app" minSdk = 21 targetSdk = 34 } } dependencies { implementation("androidx.appcompat:appcompat:1.6.1") implementation("androidx.webkit:webkit:1.8.0") }

3. Настройка AndroidManifest.xml

<!-- Разрешения --> <uses-permission android:name="android.permission.INTERNET" /> <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" /> <!-- Настройка сети для HTTP --> <application android:networkSecurityConfig="@xml/network_security_config" android:usesCleartextTraffic="true"> <activity android:name=".MainActivity"> <intent-filter> <action android:name="android.intent.action.MAIN" /> <category android:name="android.intent.category.LAUNCHER" /> </intent-filter> </activity> </application>

🌐 Деплой на сервер

🖥️ Настройка сервера

1. Подготовка сервера (Ubuntu)

# Обновление системы sudo apt update && sudo apt upgrade -y # Установка Python sudo apt install python3 python3-pip python3-venv -y # Установка Nginx sudo apt install nginx -y # Настройка фаервола sudo ufw allow 22 sudo ufw allow 80 sudo ufw allow 443 sudo ufw enable

2. Деплой приложения

# Клонирование проекта git clone https://github.com/your-username/sugdhotel.git cd sugdhotel # Настройка виртуального окружения python3 -m venv venv source venv/bin/activate pip install -r requirements.txt # Запуск через systemd sudo nano /etc/systemd/system/sugdhotel.service

3. Systemd сервис

# /etc/systemd/system/sugdhotel.service [Unit] Description=Sugdhotel FastAPI Application After=network.target [Service] Type=simple User=www-data WorkingDirectory=/path/to/sugdhotel Environment=PYTHONPATH=/path/to/sugdhotel ExecStart=/path/to/sugdhotel/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000 Restart=always [Install] WantedBy=multi-user.target

🔧 Решение проблем

❌ Проблема: Сервер не запускается

Решение:

  • Проверьте, что порт 8000 не занят: netstat -tulpn | grep 8000
  • Убедитесь, что все зависимости установлены
  • Проверьте логи: python main.py

⚠️ Проблема: WebSocket не работает

Решение:

  • Проверьте URL WebSocket: ws://your-domain.com/ws/chat
  • Убедитесь, что Nginx проксирует WebSocket
  • Проверьте настройки CORS

⚠️ Проблема: Email не отправляется

Решение:

  • Проверьте правильность email и пароля приложения
  • Для Gmail используйте пароль приложения
  • Проверьте настройки SMTP сервера
